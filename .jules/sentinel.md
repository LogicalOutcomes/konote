## 2025-03-01 - Missing rate limits on unauthenticated endpoints
**Vulnerability:** A missing `@ratelimit` decorator on the unauthenticated `password_reset_request` view. The code explicitly mentioned that the endpoint should be rate-limited in the docstring, but the actual implementation of the rate limit via Django decorator was missing. This would allow an attacker to send unauthenticated requests infinitely and cause an email spam DoS or impact quotas.
**Learning:** Even when security controls are well documented in docstrings, they can be accidentally omitted during refactoring or implementation. Unauthenticated endpoints which invoke external services (like sending emails or generating tokens) are particularly vulnerable when rate-limiting decorators are left off.
**Prevention:** Unauthenticated views that send out emails or perform expensive operations must be audited specifically for missing `@ratelimit` decorators, making sure that what is written in comments aligns with what is executed in code.
